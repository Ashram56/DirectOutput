4 \section installation_requirements Requirements
5 
6 The Directoutput framework relies on the B2S-Server which has been developed by Herweh/Stefan. The B2S-Server implements a plugin interface which can load and execute plugins. This framework has been developed as such a plugin.
7 
8 Therefore you must first install and configure Herwehs B2S-Server before you can install the DirectOutput framework. You can download the B2S-Server from <a href="http://vpuniverse.com/forums/files/file/2042-b2s-backglass-server/">vpuniverse.com</a> or <a href="http://www.vpforums.org/index.php?app=downloads&showfile=7426">VPForums</a>.

18 \section installation_installation Installation
19 
20 Hosting applications which can use the DirectOutput framework, implement plugin interfaces which will automatically detect, load and use the library if it is correctly installed.
21 
22 The DirectOutput framework can be installed in two ways:
23 
24 * Directly within the directory structure of the hosting application (e.g. B2S.Server). This will require only minimal effort when installing, but it might lead to duplicate installations if DOF is used in more than one hosting application.
25 * In any other directory of your system. This will require a little more work during installation, because shortcuts from the hosting applications plugin directory to the directoy containg DOF have to be added. This installation pattern will all all applications which support DOF to use the same installation, so no duplicate installtions are required. This is the recommanded installtion pattern.
26 
27 \subsection installation_installation_hostappdir Installation within hosting application directory structure
28 
29 To install DOF withing the directory structure of the application using the framework (e.g. B2S.Server), please do the following:
30 
31 * In the installation directory of the hosting application, create a directoy called _plugins_.
32 * Inside the newly created _plugins_ directory, create a subdirectory called _DirectOutput_ (any other name will be ok as well).
33 * Unzip the contents of the zip-file containg the framework to the newly created _DirectOutput_ directory.
34 * If using platforms like Vista or Win7, the installed files might be blocked by UAC. To unblock the files please read: \ref installation_unblockauc
35 
36 If this has been done correctly, the hosting application will automatically detect, load and use the framework.
37

44 \subsection installation_installation_owndir Installation in own directory (Recommended)
45 
46 To avoid duplicate installations of the DirectOutput framework, the applications using the framework as a plugin can follow windows shortcuts to the directory containging DOF.
47 
48 To install DirectOutput like this, please do the following:
49 * Create a directory for you DirectOutput installtion (e.g. C:\\DirectOutput). The name and the path to this directory can be anything you like, but it might be a clever idea to have DirectOutput in the directory name.
50 * Unzip the contents of the zip-file containg the framework to the newly created directory.
51 * If using platforms like Vista or Win7, the installed files might be blocked by UAC. To unblock the files please read: \ref installation_unblockauc
52 
53 To allow the hosting application (e.g. B2S.Server) to detect the DirectOutput framework, do the following:
54 * Go to the directory containing the files of the hosting application (For B2S.Server this is typically the VP tables folder).
55 * In this directory create a subdirectory named _plugins_.
56 * In the newly created _plugins_ directory, create a windows shortcut pointing to the directory containg the DirectOutput framework. This can be done by right clicking in the _plugins_ directory and selecting New -> Shortcut in the context menu.

67 \section installation_unblockauc Unblock the DLLs
68 
69 On platforms like Win7 or Vista you might need to unblock the files of the DirectOutput framework, before it can be recognized by hosting applications (e.g. B2S.Server).
70 
71 Please exceute the following procedure for all dll and exe files of the DirectOutput installtion:
72 * Right click the DLL file and select _Properties_ in the context menu.
73 * Select the _General_ tab in the properties window.
74 * If you find the text _"This file came from another computer and might be blocked to help protect this computer" or somthing similar on this tab, click the _Unblock_ button. This should fix the issue.
75 
76 If the text about the file coming from another computer does not exist, everthing should be fine.

78 \section installation_b2sserverconfig B2S.Server Configuration
79 
80 To enable the plugin support in the B2S.Server (DirectOutput is a B2S.Server plugin) you need to add the ArePluginsOn option to your B2STableSettings.xml file (In the Table directory). The result should resemble to following example:
81 ~~~~~~~~~~~~~~~{.xml}
82 <B2STableSettings>
83  ....
84  <ArePluginsOn>1</ArePluginsOn>
85  ...
86 </B2STableSettings>
87 ~~~~~~~~~~~~~~~
88 
89 \warning This block does not go to the end of the B2STableSettings.xml file. You really only have to put the line containing _ArePluginsOn_ into the file.

93 \section installation_visualpinballcorevbs Visual Pinball core.vbs Adjustment
94 
95 If you have used the <a target="_blank" href="http://www.hyperspin-fe.com/forum/showthread.php?10980-Tutorial-How-to-config-Ledwiz-PacDrive">VBScript solution</a> to control your LedWiz before you will have to _REMOVE_ the line loading ledcontrol.vbs from your core.vbs.
96 
97 Depending on your version of core.vbs this line might looks like one of the following two:
98 ~~~~~~~~~~~~~~~{.vbs}
99 ExecuteGlobal GetTextFile("ledcontrol.vbs")
100 ~~~~~~~~~~~~~~~
101 
102 ~~~~~~~~~~~~~~~{.vbs}
103 LoadScript("ledcontrol.vbs"):Err.Clear
104 ~~~~~~~~~~~~~~~
105 
106 If you dont remove the mentioned statement you'll run into trouble since the framework and the ledcontrol.vbs solutions will run simultaneously!

108 \section installation_tableconfig Table Configuration
109 
110 Tables using the DirectOutput framework resp. the B2S-Server have to instanciate the B2S.Server instead of the Pinmame.Controller.
111 Replace the following line in the table scripts of the tables you want to use the B2S.Server and the DirectOutput framework
112 
113 ~~~~~~~~~~~~~~~{.vbs}
114 Set Controller = CreateObject("VPinMAME.Controller")
115 ~~~~~~~~~~~~~~~
116 
117 with
118 
119 ~~~~~~~~~~~~~~~{.vbs}
120 Set Controller = CreateObject("B2S.Server")
121 ~~~~~~~~~~~~~~~
122 
123  For EM tables it is also necessary to a some statements to send data about the table element statuses to DirectOutput and B2S.Server.
124 
125 
126 Please read the page on \ref tableconfig for more detailed information on this topic.

128 \section installation_configuration Configuration
129 
130 The DirectOutput framework supports several configuration options:
131 
132 * __Auto configuration__ which tries to detect the connected hardware and the files containing the table definitions automatically. Auto configuration is using the legacy ledcontrol.ini file(s) for the table definitions if available.
133 * __XML config files__ are the best way to configure the framework. They allow to configure every detail of the framework in detail.
134 
135 \subsection installation_autoconfiguration Using Auto configuration
136 
137 If correctly installed the DirectOutput framework can configure itself automatically. This means that you dont need to create any of the config files mentioned in the sections below to get started.
138 
139 Auto configuration will automatically detect the LedWiz units connected to your system and try to lookup the ledcontrol.ini file(s) with table configurations in the following locations:
140 
141 - Table directory
142 - Config directory of DirectOutput
143 - Directory containing the DirectOutput.dll
144 
145 Up to 16 LedWiz units are supported, which can be controlled by numbered ledcontrol.ini files. For LedWiz nr. 1 the file ledcontrol.ini will be loaded, for LedWiz numbers 2-16 files ledcontrol{LedWizNumber}.ini (e.g. ledcontrol2.ini) will be loaded.
146 
147 To create your own lecontrol.ini files it is best to use the <a target="_blank" href="http://vpuniverse.com/ledwiz/login.php">LedWiz Config Tool</a>.
148 
149 At the moment only LedWiz units are automatically detected. If you are using other output controllers you will have to create a cabinet configuration file, specifying the output controllers and some matching LedWizEquivalent toys, to allow the use of ledcontrol files.
150 
151 \subsection installation_ledcontrolini Using LedControl.ini files
152 
153 The DirectOutput framework can use one or several classical LedControl.ini files to configure tables. If no other table config files are found, the framework will try to load the ledcontrol.ini file(s). By deafult the framework tries to find the ledcontrol files specified in the Global Config file. If global config does not specify ledcontrol file(s) the framework will search for ledcontrol file(s) in the following directories:
154 
155 - Table directory
156 - Config directory of DirectOutput
157 - Directory containing the DirectOutput.dll
158 
159 Please check the page on \ref ledcontrolfiles for details.
160 
161 \subsection installation_configfiles Using configuration files
162 
163 The auto configuration function of the framework does only support some basic functionality of the framework. If you want to unleash the full power of DirectOutput, you can venture into the more detailed configuration options.
164 
165 \subsubsection installation_globalconfig Global Configuration
166 
167 The global configuration specifies some global settings for the framework, like the places where cabinet and table configurations are looked up.
168 
169 If the framework is called through the B2S.Server Plugin it will search for a file named _GlobalConfiguration_b2SServer.xml_ in the following places:
170 - A directory named _config_ withing the directory of the DirectOutput.dll.
171 - A shortcut named _config_ to a directory.
172 - Table directory.
173 - Directory of the DirectOutput.dll.
174 
175 If no global configuration file can be found, the framework tries to configure itself by detecting LedWiz units and searching ledcontrol.ini file in the table directory, the config directory of the DirectOutput framework or the dll directory of the framework (see section on auto configuration above). In this mode, the framework behaves like the vbscript solution.
176 
177 To create your own global configuration, you can either use the built in global configuration editor or write a XML-file with the necessary information yourself.
178 
179 Please read the page on \ref global config for a detailed explanation of the settings.
180 
181 \subsubsection installation_cabinetconfig Cabinet Configuration
182 
183 The cabinet configuration specifies the output controllers (e.g. Ledwiz) and toys (e.g. contactors and RGB leds) in your cabinet.
184 
185 Typically the cabinet configuration file will be named cabinet.xml and reside in the config directory of the DirectOutput framework. Be sure to configure a file pattern in the global configuration pointing on your own cabinet configuration file.
186 
187 Please read the page on \ref cabinetconfig for more information.
188 
189 \subsubsection installation_tableconfig Table Configuration
190 
191 The table configuration specifies the elements on a pinball table (e.g. solenoids or lamps) and the effects assigned to the table elements.
192 
193 Please read the page on \ref tableconfig for details.
