6 \section contribute_bugs Report bugs and issues
7 
8 Please report bugs and issues in the DirectOutput framework!
9 
10 If you report a bug, please include as much information as possible. A simple "DirectOutput doenst work on my system" message will not help much. The following information might be helpfull to track down and fix bugs:
11 
12 - Information about your system (operating system, connected hardware).
13 - Used software (DirectOutput version, B2S Serverversion, Visual Pinball version, Pinmame version and so on).
14 - Exception message/s (most exception messages are recorded on the log files of Directoutput).
15 - Table in which the bug occured (if more than one table version exists, dont forget to include the version number or a link to the table).
16 
17 Bugs and issues can be reported as follows:
18 
19 - The prefered way of bug reporting is the <a target="_blank" href="https://github.com/DirectOutput/DirectOutput/issues/new">create a issue</a> on the <a target="_blank" href="https://github.com/DirectOutput/DirectOutput/">Github page</a> of the DirectOutput framework. Creating a new issue on GitHub requires a login on Github, but creating accounts on Github is easy to create and free.
20 - If you are feeling really uncomfortable about GitHub, you can also report bug through <a href="">VPForums</a>.
21 
22 Important!
23 ----
24 
25 
26 - Please check the reported issues on <a target="_blank" href="https://github.com/DirectOutput/DirectOutput/issues?state=open">GitHub</a> and VPForums, to find out if a specific issue has already been reported, before creating a new issue. Thanks!
27 - Before reporting a issue, please check if you are using the latest version of the framework.

36 \section contribute_help Share your knowledge

38 Some users might have problems installing, configuring and using the framwork. If you already feel familiar with the framework please consider, helping other users by ansering their questions in the various virtual pinball forums. This will free up resources on the development side!

41 \section contribute_scripting Create scripts
42 
43 The DirectOutput framework can be extended using C# script files. Using scripts you can create new effects, toys and output controllers.
44 
45 Create more effects
46 --
47 
48 The DirectOutput framework can be extended easily with more effects. If you know a bit about C# (not that hard to learn), you might want to start implementing some cool effects for the framework. More information on effect creation and samples can be found on documentation page on effects.
49 
50 Add more and better toys
51 --
52 
53 Toys represent all the cool gadgets that you have installed in your cabinet. If you find that a toy is missing or requires a better implementation, you are very welcome to contribute some code using the built in scriptingf capabilities. More information on the implemention of toy scripts can be found on the documentation page on toys.
54 
55 Add more output controllers
56 --
57 
58 If you are using a output controller which is not yet supported by the framework, it would be great if you add support for it. You can find more information on the page on output controllers.

61 \section contribute_source Contribute to documentation
62 
63 If you think that something important is missing in the docu on DirectOutput, just add whatever is missing to the documentation.
64 
65 - If you have forked the source code (see the docupage on source code for details), you can add you contribution directly to the DirectOutput project and file pull request on Github to make it available for the project.
66 - You are also welcome to add documention pages, if you are not working with the source code. Just forward your contribution to one of the <a target="_blank" href="https://github.com/DirectOutput?tab=members">members of the DirectOutput project</a> so it can be included in the project.

71 \section contribute_source Contribute to the source code
72 
73 The source code of the DirectOutput framework is freely available on <a target="_blank" href="https://github.com/DirectOutput/DirectOutput/">GitHub</a>.
74 
75 You are more than welcome to download, analyze, modify, improve or extend the code. The best way to do this is to fork the source code (using Git), so you can easily upload your changes back to GitHub. More information can be found on the documention page on Source Code.
