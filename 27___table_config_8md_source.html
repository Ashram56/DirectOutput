4 \section tableconfig_introduction Introduction
5 
6 The table configuration defines the table elements (e.g. solenoids or lamps), the configuration of the effects and the assigment of the effects to the table elements. The assigned effects are triggered if the value of a table element changes (typicaly when new data from Pinmame is received).
7 
8 In addition there is also a collection of static effects which are triggered when the framework is starting up. Use the static effects to control toys which dont have to change their state during game play (e.g. set illuminated flipper button to a specific color).
9 
10 The framework does also allow to mix the configuration in a table config file with configurations loaded from ledcontrol files. This will allow you to use configurations from the <a href="http://vpuniverse.com/ledwiz/login.php">LedWiz Config Tool</a> for commonly used gadgets and to supplement this with your own config.
11 
12 \section tableconfig_configfilestructure Table config file
13 
14 Table configurations are stored as XML-files. The basic structure of a table configuration file contains the following sections:
15 
16 * __Effects__ configures all effects for a table. These effects can be assigned to table elements or to the static effects.
17 * __TableElements__ defines the table elements with their type (e.g. solenoid or lamp) and number and also the assignment of effects to the table elements.
18 * __AssignedStaticEffects__ defines the static effects for the table.
19 * __TableName__ contains the name of the pinball table.
20 
21 The basic structure of a table configuration looks as follows:
22 
23 ~~~~~~~~~~~~~{.xml}
24 <?xml version="1.0"?>
25 <Table xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
26  <TableName>Name of the table</TableName>
27  <TableElements>
28  ... Table element definitions ...
29  </TableElements>
30  <Effects>
31  ... Effects configuration ...
32  </Effects>
33  <AssignedStaticEffects>
34  ... Static effects assignments ...
35  </AssignedStaticEffects>
36 
37  <AddLedControlConfig>true/false</AddLedControlConfig>
38 
39 </Table>
40 ~~~~~~~~~~~~~
41 
42 \subsection tableconfig_configfilestructuretableelements Table elements section
43 
44 The TableElements section contains the definition of the table elements (solenoids, lamps and so on) of the pinball table. Table elements are configured using TableElement sections. A TableElements secation can contain any number of TableElement sections.
45 
46 The definition of a table element consists of the type and the number of the table element. A name for the element can be specified in addition. The name has no technical relevance, but can be useful for informational purposes and configuring a table will for sure be easier if the table elements are named.
47 
48 For every table element one or several effects can be assigned. A effect assignment consists of the name of the effect and a number which defines the order in which the effects are triggered. If the order of the effects doesnt matter, you can also use the same order number for every assigned effect.
49 
50 On initialization of the framework the names of the assigned effects are resolved into references to the actual effect objects. If a name cant be resolved, the effect assignmenet will be disabled.
51 
52 A typical table element definition looks as follows:
53 
54 ~~~~~~~~~~~~~{.xml}
55  <TableElement>
56  <TableElementType>Solenoid</TableElementType>
57  <Number>4</Number>
58  <Name>Right slingshot</Name>
59  <AssignedEffects>
60  <AssignedEffectOrder>
61  <EffectName>Fire Right Bottom Contactor</EffectName>
62  <Order>1</Order>
63  </AssignedEffectOrder>
64  <AssignedEffectOrder>
65  <EffectName>Flash Right Backboard Led</EffectName>
66  <Order>2</Order>
67  </AssignedEffectOrder>
68  </AssignedEffects>
69  </TableElement>
70 ~~~~~~~~~~~~~

72 \subsection tableconfig_configfilestructurestaticeffects Assigned static effects section
73 
74 The AssignedStaticEffects contains the assignment of the static effects which are triggered when the system starts up. The static effects are defined by using AssignedEffectOrder sections.
75 
76 The AssignedEffectOrder sections consist of the name of the assigned effect and a number specifying the order in which the static effects are triggered.
77 
78 On initialization of the framework the names of the assigned effects are resolved into references to the actual effect objects. If a name cant be resolved, the effect assignmenet will be disabled.
79 
80 A typical AssignedEffectOrder might looks like this:
81 
82 ~~~~~~~~~~~~~{.xml}
83  <AssignedEffectOrder>
84  <EffectName>Set Flipperbutton to Red</EffectName>
85  <Order>1</Order>
86  </AssignedEffectOrder>
87 ~~~~~~~~~~~~~

90 \subsection tableconfig_configfilestructureeffects Effects section
91 
92 The effects section contains the configurations for all table effects. Effects configured in this section can be referenced by their name in the static effects section and in the assigned effect sections of the various table elements.
93 
94 The Directoutput framework contains some built in effects and more effects can be added through scripting (see scripting page for details).
95 
96 At the time of writting the following built in effects exist in the framewwork:
97 
98 \subsubsection tableconfig_configfilestructureeffectsbasicanalogeffect BasicAnalogEffect
99 
100 The BasicAnalogEffect can be used to control toys which are using analog outputs (e.g. lamps). These toys have to implement the IAnalogToy interface.
101 
102 This effect will set the value of the assigned toy to the values defined in ValueOn and ValueOff depending on the value of the table element triggering the effect. Using the effect as a static effect is also possible. The effect will set the toys value to ValueOn if it as called as a static effect.
103 
104 A typical config for this effect type might looks as follows:
105 
106 <BasicAnalogEffect>
107  <Name>Effect name</Name>
108  <AnalogToyName>Name of a analog toy</AnalogToyName>
109  <ValueOn>255</ValueOn>
110  <ValueOff>0</ValueOff>
111 </BasicAnalogEffect>
112 
113 * __Name__ is the name of the effect. What else?
114 * __AnalogToyName__ is the name of toy a implementing the IAnalogToy interface. The toy has to be configured in the cabinet.
115 * __ValueOn__ is the value which will be set for the toy, if the effect is triggered by a table element having a value which is not equal 0. The value will also be used the set the value of the referenced toy, if the effect is assigned as a static effect.
116 * __ValueOff__ is the value which will be set for the toy, if the effect is triggered by a table element having a value of 0.
117 
118 \subsubsection tableconfig_configfilestructureeffectsbasicdigitaleffect BasicDigitalEffect
119 
120 The BasicDigitalEffect is used to control toy which are using digital outputs (e.g. contactors). The toys have to implement the IDigitalToy interface.
121 
122 This effect sets the state of a toy implementing the IDigitalToy interface depending on the value of the table element triggering the effect. A table element value of 0, will set the toy state to false/off, any other value will set the toys state to true/on. If assigned as a static effect, the state of the toy will always be set to true/on.
123 
124 Here is a typical BasicDigitalEffect configuration:
125 
126 <BasicDigitalEffect>
127  <Name>Effect name</Name>
128  <DigitalToyName>Name of a toy</DigitalToyName>
129 </BasicDigitalEffect>
130 
131 * __Name__ is as usual the name of the effect.
132 * __DigitalToyName__ is the name of the assigned toy. The assigned toy has be configured in the cabinet and must to implement the IDigitalToy interface.
133 
134 \subsubsection tableconfig_configfilestructureeffectsbasicrgbeffect BasicRGBEffect
135 
136 The BasicRGBEffect is used to control RGBLeds or any other toy implementing the IRGBToy interface. Depending on the vaÃ¶lue of the table element triggering the effect, the IRGBToy will be set to the configured color or to off. A table element value of 0 will turn of the IRGBToy, a value not equal 0 will set the toy to the specified color.
137 
138 If the effect is assigned as a static effect it will set the assigned toy to to specified color.
139 
140 <BasicRGBEffect>
141  <Name>Effect name</Name>
142  <RGBToyName>RGBLedName value</RGBToyName>
143  <Color>Color value (e.g. #00ff00 for blue)</Color>
144 </BasicRGBEffect>
145 
146 * __Name__ of the effect.
147 * __RGBToyName__ is the name of the assigned RGB toy. This toy must implement the IRGBToy interface.
148 * __Color__ defines color which is set for the toy, if table element triggering the effect has a value which not equal 0. The color can be specified as a hexadecimal color definition (e.g. \#ff0000 for red), comma separated color (e.g. 0,255,0 for green) or color name as defined in the cabinet colors section.
149 
150 \subsubsection tableconfig_configfilestructureeffectslisteffect ListEffect
151 
152 The ListEffect is a effect which triggers a list of other effects assigned to the list effect. The data of the table element which has triggered the effect is passed on to the effects assigned to the list effect.
153 Used as a static effect the ListEffect will trigger all assigned effect as well.
154 
155 
156 <ListEffect>
157  <Name>Effect name</Name>
158  <AssignedEffects>
159  <AssignedEffectOrder>
160  <EffectName>Name of the first effect</EffectName>
161  <Order>1</Order>
162  </AssignedEffectOrder>
163  <AssignedEffectOrder>
164  <EffectName>Name of a second effect</EffectName>
165  <Order>2</Order>
166  </AssignedEffectOrder>
167  </AssignedEffects>
168 </ListEffect>
169 
170 * __Name__ is the name of the effect. Hard to guess?
171 * __AssignedEffects__ section contains any number of ordered effect assignments.
172 * __EffectName__ contains the name of the assigned effect.
173 * __Order__ is a number specifying the order in which the assigned effects are executed.

176 \subsection tableconfig_configfilestructureaddledcontrolconfig AddLedControlConfig section
177 
178 This section does only contrin true or false.
179 
180 True will tell the system to supplement the table config which has been loaded from the config xml with configurations from ledcontrol file(s) if available.
181 
182 False will not add configs from ledcontrol file(s).
