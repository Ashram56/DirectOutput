Detailed Description

The RGBToyGroup can be used to build groups of toys implementing the IRGBToy interface.
Since the RGBToyGroup does also implement the IRGBToy interface, it is possible to build nested RGBToyGroups (RGBToyGroups containging other RGBToyGroups).
The implemented IRGBToy interface does also allow to treat a RGBToyGroup like any other IRGBToy, but the methods and properties will allways act on all assigned IRGBToy objects (e.g. SetColor will set the color of all assigned IRGBToy objects).
RGBToyGroup does also implement a indexer property allowing direct access to all assigned IRGBToy objects, including those assigned in nested RGBToyGroup objects (e.g. if a group has 5 RGBLed object and 1 RGBToyGroup object containing 3 RGBLeds specified in the AssignedToys property, the indexer will enumerate all 8 RGBLed objects directly.

Note
: The behaviour of this object is somewhat different from other collections. The IRGBToy objects in the group have to be specified in the AssignedToys property and can be enumerated normaly. However if you enumerate the RGBToyGroup it will also enumerate all IRGBToy objects in AssignedToys, but contained RGBToyGroup objects will be resolved into the contained IRGBToy objects.
Warning
: Since it is possible to build nested groups, you have to be carefull not to create circular references. The RGBToyGroup does NOT check for circular references and the framework will throw a exception or even crash due to a stack overflow if circular references occur.
